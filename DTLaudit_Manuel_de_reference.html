DTLaudit — Manuel de référence

Référence descriptive de l’outil d’audit local des projets DTL.

Type : audit lecture seuleLangage : PythonVersion code : v1.1-6

Rôle du composant

DTLaudit fournit une vue de contrôle sur un projet ou une suite de projets. Il agrège des informations de structure, de documentation, de versionnement Git et de présence GitHub afin de produire un diagnostic synthétique de cohérence.

L’outil est conçu pour observer sans modifier. Il ne corrige pas les dépôts, ne nettoie pas les fichiers et ne déclénche aucune publication.

Périmètre fonctionnel

Audit projet

Le mode projet décrit un répertoire unique : présence Git, état de la branche, remote, tags, README, documentation et fichiers significatifs.

Audit suite

Le mode suite compare les sous-projets directs d’un dossier racine et met en évidence les différences de structure entre projets.

Audit GitHub

Quand GitHub CLI est disponible, DTLaudit enrichit le rapport avec des informations distantes : visibilité, release récente, pull requests et issues ouvertes.

Architecture interne

DTLaudit.py

Le script principal regroupe la découverte des projets, la collecte des fichiers, la lecture des informations Git, l’appel optionnel à GitHub CLI, la sérialisation et la génération des rapports.

BlocDescription
DécouverteIdentifie les dossiers candidats selon la présence de Git, Python, scripts Windows ou README.
Collecte GitLit branche courante, remote, tags, état local et historique minimal.
Audit documentaireVérifie README, guides utilisateur et manuels de référence.
RenduProduit HTML par défaut, avec sorties JSON et texte possibles.

Données observées

  • Fichiers et dossiers non ignorés du projet.
  • Dossiers générés comme build, dist et cachés Python.
  • Fichiers volumineux dépassant le seuil de 5 Mo.
  • État Git local et métadonnées GitHub disponibles.
  • Présence de documents README, guides et références.

Modèle de sortie

Le rapport HTML est la sortie principale. Il contient une synthèse globale, un bloc détaillé par projet, des observations notables et une matrice de fichiers permettant de repérer les écarts entre projets.

La sortie JSON conserve une structure exploitable par un tableau de bord local ou un traitement automatisé. La sortie texte sert à la consultation rapide.

Sécurité et non-altération

DTLaudit est un outil en lecture seule. Les commandes externes sont limitées à la lecture d’informations Git et GitHub. Aucun fichier audité n’est modifié par le processus d’audit.

Limites connues

  • Les métadonnées GitHub dépendent de l’installation et de l’authentification de `gh`.
  • La détection des projets reste volontairement simple et s’appuie sur des marqueurs locaux.
  • Les artefacts générés suivis par Git peuvent produire du bruit dans les constats.
